What Are Claude Code and Ollama?

Claude Code is Anthropic's terminal-based AI coding agent. It reads your codebase, plans changes, edits files across your project, runs shell commands, executes tests, and handles git workflows — all from the command line. Unlike IDE plugins, Claude Code operates directly on your filesystem and understands your entire project structure through on-demand file reading. It supports multi-step reasoning, tool calling, and persistent project memory via CLAUDE.md files.

Ollama is a free, open-source runtime for local large language models (MIT license, 171,000+ GitHub stars as of May 2026). It handles model download, quantization, GPU allocation, and serves an API on localhost:11434. Since version 0.14.0 (January 2026), Ollama natively exposes an Anthropic Messages API-compatible endpoint — meaning Claude Code can talk to local models directly, with no proxy or translation layer required.

"Claude Code Ollama" is not a single product. It is the combination of these two tools: the Claude Code CLI configured to route its API calls to Ollama's local endpoint instead of Anthropic's cloud. The result is a fully functional AI coding agent that runs on your hardware at zero API cost.

Why Run Claude Code Locally?

Using Claude Code with Anthropic's API costs approximately $150–$250 per developer per month for regular use, according to Anthropic's own cost documentation (docs.anthropic.com/en/docs/claude-code/costs). Running it locally through Ollama eliminates API charges entirely. Your only cost is the hardware you already own and the electricity to power it.

There are three additional benefits:

  • Privacy: Your code never leaves your machine. All inference happens locally — no data transmitted to external servers.
  • No rate limits: Cloud API plans impose message caps (5x or 20x Pro on Max plans). Local inference has no usage limits.
  • Offline capability: Once models are downloaded, the entire stack works without internet access.
Important context: When I first set this up on a MacBook Pro M3 Max (48GB), pulling a Qwen 3.5 27B model for the first time downloaded approximately 16 GB and took 22 minutes on a 200 Mbps connection. Subsequent cold starts load the model into memory in about 45 seconds. Keep these times in mind when planning your first session.

Step 1: Install Ollama

Download Ollama from ollama.com for your operating system. On macOS, the recommended method is Homebrew:

brew install ollama
brew upgrade ollama   # ensure 0.19+ for MLX support

On Linux, use the install script:

curl -fsSL https://ollama.com/install.sh | sh

On Windows, download the installer from ollama.com (native ARM64 support included).

After installation, start the Ollama server and verify it is running:

ollama serve

Open a second terminal and run:

curl http://localhost:11434/api/tags

You should see a JSON response — empty if you have no models downloaded yet, which is expected at this stage:

{"models":[]}

If you get a connection refused error, the Ollama background service may not be running. Start it manually with ollama serve.

Step 2: Pull a Coding Model

Model selection determines the quality, speed, and memory tradeoff. Choose based on your available RAM. These are the verified recommendations for coding workloads as of May 2026:

qwen3-coder:14b ~9 GB 12 GB total Coding-specialized, reasonable quality ollama pull qwen3-coder:14b

Pull your chosen model (the 9B or flash models are the best starting point for 16 GB machines):

ollama pull qwen3.5:9b

The download happens once. After the model is cached locally, subsequent loads are instant. On my test machine, pulling qwen3.5:9b downloaded 5.0 GB and completed in 7 minutes on a 100 Mbps connection.

Step 3: Install Claude Code

Claude Code installs as a standalone binary. On macOS and Linux, use the official installer:

curl -fsSL https://claude.ai/install.sh | bash

On Windows (PowerShell):

irm https://claude.ai/install.ps1 | iex

The npm-based installation (npm install -g @anthropic-ai/claude-code) is deprecated — use the native installer linked above.

Verify the installation:

claude --version
No subscription required: You do not need an Anthropic subscription to install or run Claude Code. The CLI binary is free. You only need a subscription if you route requests to Anthropic's cloud API. With Ollama, all inference runs locally — no API key or subscription is needed.

Step 4: Connect Claude Code to Ollama

There are three methods. Use the simplest one that fits your workflow.

Method A: ollama launch (Ollama v0.15+, recommended)

ollama launch claude --model qwen3.5:9b

This single command automatically sets ANTHROPIC_AUTH_TOKEN, ANTHROPIC_BASE_URL, and ANTHROPIC_API_KEY, then starts Claude Code pointed at your local Ollama instance. It walks through model selection on first run. This is the easiest method.

Method B: Environment variables (manual)

Add these to your shell profile (~/.zshrc, ~/.bashrc, or ~/.config/fish/config.fish):

export ANTHROPIC_BASE_URL="http://localhost:11434"
export ANTHROPIC_AUTH_TOKEN="ollama"
export ANTHROPIC_API_KEY=""

Then launch Claude Code in your project directory:

cd /path/to/your/project
claude --model qwen3.5:9b
Critical detail: ANTHROPIC_API_KEY must be explicitly set to an empty string — not missing, not unset. If it is unset, Claude Code falls back to cached credentials from a previous Anthropic login and will attempt to reach the cloud API instead of your local Ollama instance. If you were previously logged in, run /logout inside a Claude Code session first.

Method C: Persistent config file

Create or edit ~/.claude/settings.json:

{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:11434",
    "ANTHROPIC_AUTH_TOKEN": "ollama",
    "ANTHROPIC_API_KEY": ""
  }
}

Now every claude session automatically connects to Ollama. Switch models with the --model flag:

claude --model qwen3.5:27b
claude --model glm-4.7:flash

Step 5: Test Your Setup

Navigate to a project directory and give Claude Code a real task:

cd ~/projects/my-app
claude --model qwen3.5:9b

Inside the Claude Code session, try:

/status

This should show the project structure and confirm the agent is connected. Then give it a task:

Explain the architecture of this project. List all imports and identify any circular dependencies.

If you see a response generated by your local model, the setup is working. Expect the first prompt to take 30–60 seconds — the model loads into GPU memory on first use (cold start). Subsequent prompts are much faster.

Keep models loaded: Set OLLAMA_KEEP_ALIVE=30m (or -1 to keep loaded indefinitely) to avoid repeated cold starts during a coding session. Add this to your shell profile: export OLLAMA_KEEP_ALIVE=30m.

Real-World Performance: What to Expect

Local models are not free in every sense — they trade API costs for hardware and speed constraints. Here are measured benchmarks from my test setup (MacBook Pro M3 Max, 48GB, Ollama v0.19.0):

Model Inference Speed Cold Start Response Quality Context Window
qwen3.5:4b ~112 tok/s ~15s Basic correctness, unreliable tool calling 32K
qwen3.5:9b ~78 tok/s ~22s Decent coding, reliable single-file edits 32K
glm-4.7:flash ~85 tok/s ~20s Good speed, large context (128K) 128K
qwen3.5:27b ~58 tok/s ~45s Strong reasoning, multi-file refactoring 32K
gpt-oss:20b ~65 tok/s ~35s Best tool calling at this tier 32K

For comparison, Claude 3.5 Sonnet via Anthropic's API delivers responses at approximately 80–120 tok/s with near-instant cold start and substantially better reasoning quality — but at $3 per million input tokens and $15 per million output tokens.

One real limitation I discovered during testing: Claude Code's tool-calling loop generates approximately 3–5 round trips per request (for planning, file reading, editing, and verification). With a local 8B model at 78 tok/s, a single multi-step edit takes 90–180 seconds. The same task with cloud Sonnet completes in 15–30 seconds. This is not a bug — it is the inherent speed difference between local quantization and cloud-scale inference.

Who Should NOT Use This Setup

  • You work on massive monorepos (100,000+ files): Claude Code reads files on demand, but local models struggle to maintain coherence across very large codebases. The 32K context window of most open models is insufficient for deeply nested project structures.
  • You need production-grade analysis: If you are refactoring a payment system, writing safety-critical code, or handling compliance-sensitive logic, local open-weight models make more reasoning errors than Claude Sonnet or Opus. My testing showed approximately 15–20% higher error rates on complex multi-step reasoning tasks compared to cloud Claude.
  • You have less than 16 GB of RAM: 8B models on 8 GB machines are extremely slow (CPU fallback, 5–15 tok/s) and produce unreliable tool calls. You will have a better experience with cloud-based free tiers.
  • You need instant responses: The 30–60 second cold start and 90–180 second multi-step edits mean this setup is not suitable for rapid interactive coding sessions. Use cloud Claude or GitHub Copilot for that workflow.

Frequently Asked Questions

Is Claude Code free to use without an Anthropic subscription?
Yes. The Claude Code CLI binary is free to download and install. You only need an Anthropic subscription (Pro $20/mo, Max $100–$200/mo) if you route requests to Anthropic's cloud API. When using Ollama for local inference, no subscription is required — all computation runs on your hardware.
Does Ollama's Anthropic API compatibility require a proxy?
No, as of Ollama v0.14.0 (released January 2026), Ollama natively exposes an Anthropic Messages API-compatible endpoint at localhost:11434. Claude Code can connect directly — no proxy, no translation layer. Set ANTHROPIC_BASE_URL=http://localhost:11434 and ANTHROPIC_AUTH_TOKEN=ollama, and it works.
What is the best coding model for 16 GB RAM?
For 16 GB machines (typical MacBook Air or Pro base model), qwen3.5:9b (~5 GB download, ~7 GB RAM) and glm-4.7:flash (~5.5 GB download, ~8 GB RAM) are the sweet spots. The GLM model offers a 128K context window, which is useful for large codebases. For coding-specific workloads, qwen3-coder:14b is also viable if you have 18+ GB of headroom.
Can I use Ollama's cloud models with Claude Code?
Yes. Ollama Cloud offers free-tier models like glm-4.7:cloud and qwen3.5:cloud with generous rate limits. Run ollama launch claude --model qwen3.5:cloud to use them. These run on Ollama's infrastructure — you get faster inference and larger context windows, but your code is sent to external servers, which defeats the privacy benefit of local inference.
How does response quality compare to Claude 3.5 Sonnet?
Local open-weight models (even 27B+ parameters) are noticeably less capable than Claude 3.5 Sonnet on complex reasoning tasks — particularly multi-step refactoring, understanding deeply nested callbacks, and generating idiomatic code with framework-specific patterns. For simple tasks (writing functions, fixing syntax errors, explaining code), the gap is small. For complex architecture decisions, the gap is substantial. Ollama's cloud models (like qwen3.5:cloud) close this gap significantly at the cost of sending data to external servers.
Can I switch between local and cloud models in the same session?
Yes. Use the /model command inside a Claude Code session to switch between models. You can also start a session with a different model by passing --model on the command line. If you configure ~/.claude/settings.json with Ollama as default, individual sessions can override the model with the --model flag.
Does this work on Windows?
Yes. Ollama has native Windows support (including ARM64). Claude Code installs on Windows via irm https://claude.ai/install.ps1 | iex. The same environment variable configuration works. The main limitation is GPU support — Windows with NVIDIA GPUs works well (CUDA acceleration is supported by Ollama), but DirectML support for AMD/Intel GPUs is less mature.

Final Verdict

Claude Code + Ollama is the best free option for AI-assisted coding in 2026 — if you have the hardware. It delivers zero-cost, private, offline-capable AI coding assistance with full Claude Code tooling. The tradeoff is real: local models are slower and less capable than Anthropic's cloud Claude. For developers who already own a 16 GB+ machine and value privacy or need offline capability, this combination is unmatched. For developers who need maximum reasoning quality and speed, the $20/month Claude Pro subscription remains the better investment.

The setup requires approximately 30 minutes and one-time downloads of 5–16 GB. After that, there are no recurring costs. The ollama launch claude command (Ollama v0.15+) makes the setup trivial.
